# Contributing to flutter_reactions

First of all, thank you for taking the time to contribute ❤️
Community contributions are very welcome and help make this package better for everyone.

---

## 📌 General Rules

* Please be respectful and constructive in all discussions.
* Keep changes focused and easy to review.
* Follow common Flutter/Dart best practices.
* The maintainer will review all Pull Requests and **only the maintainer has permission to merge PRs** to keep the project stable.

---

## 🧠 Before You Start

If you plan to:

* Change public API
* Add a new feature
* Refactor important logic

👉 Please create a GitHub Issue first to discuss the idea.

If you are fixing a small bug, you can open a PR directly, but an issue is still appreciated.

---

## 🚀 Creating a Pull Request

1. **Fork the repository**
2. Create a new branch from `main` or `master`
3. Install dependencies:

   ```bash
   flutter pub get
   ```
4. Make your changes
5. Format code:

   ```bash
   make format
   ```
6. Analyze code:

   ```bash
   dart analyze
   ```
7. Add tests if needed
8. Write a clear commit message
9. Push to your fork
10. Create a Pull Request targeting `dev` branch

After you submit a PR, please wait for review before making more changes unless requested.

---

## ✅ Code Style

* Follow Flutter official style guide
* Keep widgets simple and reusable
* Avoid breaking public API unless discussed first
* Add documentation for new public properties

---

## 🧪 Testing

If your change affects behavior:

* Add or update tests
* Ensure the project runs correctly

PRs without tests for important logic may be requested to add them before approval.

---

## 📚 Documentation

If you add new features:

* Update README if needed
* Add usage example

Clear examples help other developers understand the feature.

---

## 🙏 Thank You

Thank you again for helping improve flutter_reactions.
Your contribution is appreciated!
